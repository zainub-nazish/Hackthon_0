## Prompt

Create Orchestrator logic: Watch /Approved folder and trigger MCP. Add basic cron scheduling example in README (daily 8 AM briefing). Also create /Scripts/orchestrator.py skeleton for running watchers with pm2.

## Response snapshot

Created three artefacts:

1. `Scripts/orchestrator.py` (full process manager + cron scheduler):
   - `ProcessManager` class: spawns all 5 watchers as subprocesses, health-checks every 10s, restarts with exponential backoff (cap 5 min), stdout/stderr routed to Logs/<name>.{stdout,stderr}.log
   - `CronScheduler` class: minute-resolution, no external deps, daemon-thread execution, dedup per minute
   - `job_daily_briefing()`: reads Needs_Action/Plans/Pending_Approval, builds HTML digest, sends via MCPEmailClient to BRIEFING_RECIPIENT env var
   - `Orchestrator`: composes both; modes: standalone (all watchers + cron) or cron-only (for pm2 pairing)
   - CLI flags: --mode, --list, --generate-pm2
   - `generate_pm2_config()`: dynamically builds ecosystem.config.js from WATCHER_REGISTRY

2. `ecosystem.config.js` (generated by orchestrator --generate-pm2):
   - 5 watcher apps + 1 orchestrator-cron app
   - Per-watcher restart_delay and max_restarts from WATCHER_REGISTRY
   - orchestrator-cron runs in --mode cron-only so no duplicate watcher spawning

3. `README.md` (comprehensive project docs):
   - ASCII architecture diagram (signals -> watchers -> Needs_Action -> reasoning -> approval -> execution)
   - Quick start (credentials, npm build, venv, standalone run)
   - pm2 section: generate-pm2, start/list/logs/restart/stop, save+startup
   - Cron section: built-in scheduler usage, system crontab examples, pm2 cron_restart alternative
   - Approval workflow step-by-step
   - MCP email server reference with mcp.json snippet
   - Watcher reference table
   - Credentials setup guide
